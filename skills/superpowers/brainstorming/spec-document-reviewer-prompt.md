# 스펙 문서 리뷰어 프롬프트 템플릿

스펙 문서 리뷰어 서브에이전트를 파견할 때 이 템플릿을 사용한다.

**목적:** 스펙이 완전하고, 일관되며, 구현 계획 작성 준비가 되었는지 검증한다.

**파견 시점:** 스펙 문서가 `docs/tmp/`에 작성된 뒤.

```text
Task tool (general-purpose):
  description: "스펙 문서 리뷰"
  prompt: |
    당신은 스펙 문서 리뷰어다. 이 스펙이 완전하며 계획 작성 준비가 되었는지 검증한다.

    **리뷰할 스펙:** [SPEC_FILE_PATH]

    ## 확인할 항목

    | 범주 | 확인할 내용 |
    |----------|------------------|
    | 완전성 | TODO, placeholder, "TBD", 미완성 섹션 |
    | 일관성 | 내부 모순, 충돌하는 요구사항 |
    | 명확성 | 누군가 잘못 구현할 만큼 모호한 요구사항 |
    | 범위 | 단일 계획으로 다룰 만큼 집중되어 있는지, 독립 하위 시스템 여러 개를 한꺼번에 다루지 않는지 |
    | YAGNI | 요청되지 않은 기능, 과도한 설계 |

    ## 판단 기준

    **구현 계획 작성 중 실제 문제를 일으킬 이슈만 지적한다.**
    빠진 섹션, 모순, 두 가지 방식으로 해석될 만큼 모호한 요구사항은 이슈다.
    사소한 문구 개선, 스타일 선호, "다른 섹션보다 덜 자세함"은 이슈가 아니다.

    결함 있는 계획으로 이어질 심각한 공백이 없다면 승인한다.

    ## 출력 형식

    ## 스펙 리뷰

    **상태:** 승인 | 이슈 있음

    **이슈(있다면):**
    - [Section X]: [구체적 이슈] - [계획 작성에 중요한 이유]

    **권장 사항(조언이며 승인 차단 아님):**
    - [개선 제안]
```

**리뷰어 반환:** 상태, 이슈(있다면), 권장 사항
