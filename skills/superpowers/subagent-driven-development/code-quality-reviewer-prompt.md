# 코드 품질 리뷰어 프롬프트 템플릿

코드 품질 리뷰어 서브에이전트를 파견할 때 이 템플릿을 사용한다.

**목적:** 구현이 잘 만들어졌는지 검증한다. 깨끗하고, 테스트되어 있으며, 유지보수 가능해야 한다.

**스펙 준수 리뷰가 통과한 뒤에만 파견한다.**

```text
Task tool (general-purpose):
  requesting-code-review/code-reviewer.md 템플릿을 사용한다.

  DESCRIPTION: [구현자 보고서의 작업 요약]
  PLAN_OR_REQUIREMENTS: [plan-file]의 Task N
  BASE_SHA: [작업 전 commit]
  HEAD_SHA: [현재 commit]
```

**표준 코드 품질 관심사에 더해, 리뷰어는 다음을 확인해야 한다:**

- 각 파일이 잘 정의된 인터페이스와 하나의 명확한 책임을 가지는가?
- 단위가 독립적으로 이해하고 테스트할 수 있게 분해되어 있는가?
- 구현이 계획의 파일 구조를 따르는가?
- 이 구현이 이미 큰 새 파일을 만들었거나 기존 파일을 크게 키웠는가? 기존 파일 크기는 지적하지 말고, 이번 변경이 기여한 부분에 집중한다.

**코드 리뷰어 반환:** 강점, 이슈(Critical/Important/Minor), 평가
