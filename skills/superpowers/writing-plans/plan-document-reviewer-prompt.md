# 계획 문서 리뷰어 프롬프트 템플릿

계획 문서 리뷰어 서브에이전트를 파견할 때 이 템플릿을 사용한다.

**목적:** 계획이 완전하고, 스펙과 맞으며, 작업 분해가 적절한지 검증한다.

**파견 시점:** 전체 계획이 작성된 뒤.

```text
Task tool (general-purpose):
  description: "계획 문서 리뷰"
  prompt: |
    당신은 계획 문서 리뷰어다. 이 계획이 완전하며 구현을 시작할 준비가 되었는지 검증한다.

    **리뷰할 계획:** [PLAN_FILE_PATH]
    **참고할 스펙:** [SPEC_FILE_PATH]

    ## 확인할 항목

    | 범주 | 확인할 내용 |
    |----------|------------------|
    | 완전성 | TODO, placeholder, 미완성 작업, 빠진 단계 |
    | 스펙 정렬 | 계획이 스펙 요구사항을 다루는지, 큰 scope creep이 없는지 |
    | 작업 분해 | 작업 경계가 명확하고 단계가 실행 가능한지 |
    | 구현 가능성 | 엔지니어가 막히지 않고 이 계획을 따라갈 수 있는지 |

    ## 판단 기준

    **구현 중 실제 문제를 일으킬 이슈만 지적한다.**
    구현자가 잘못된 것을 만들거나 막히게 되는 문제는 이슈다.
    사소한 표현, 스타일 선호, "있으면 좋은" 제안은 이슈가 아니다.

    스펙 요구사항 누락, 모순된 단계, placeholder 내용, 실행할 수 없을 정도로
    모호한 작업처럼 심각한 공백이 없다면 승인한다.

    ## 출력 형식

    ## 계획 리뷰

    **상태:** 승인 | 이슈 있음

    **이슈(있다면):**
    - [Task X, Step Y]: [구체적 이슈] - [구현에 중요한 이유]

    **권장 사항(조언이며 승인 차단 아님):**
    - [개선 제안]
```

**리뷰어 반환:** 상태, 이슈(있다면), 권장 사항
