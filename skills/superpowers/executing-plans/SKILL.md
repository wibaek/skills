# 계획 실행

## 개요

계획을 불러오고, 비판적으로 검토하고, 모든 작업을 실행한 뒤 완료되면 보고한다.

**시작할 때 알림:** "이 계획을 구현하기 위해 executing-plans 스킬을 사용합니다."

**참고:** 서브에이전트를 사용할 수 있다면 이 스킬 대신 `superpowers:subagent-driven-development`를 사용한다.

## 프로세스

### 1단계: 계획 불러오기 및 검토

1. 계획 파일을 읽는다.
2. 계획을 비판적으로 검토한다. 질문이나 우려 사항을 찾는다.
3. 우려 사항이 있으면 시작하기 전에 사람 파트너에게 제기한다.
4. 우려 사항이 없으면 TodoWrite를 만들고 진행한다.

### 2단계: 작업 실행

각 작업마다:

1. `in_progress`로 표시한다.
2. 각 단계를 정확히 따른다. 계획은 작게 나뉜 단계로 구성되어 있다.
3. 지정된 검증을 실행한다.
4. `completed`로 표시한다.

### 3단계: 개발 완료

모든 작업이 완료되고 검증되면:

- 알림: "이 작업을 마무리하기 위해 finishing-a-development-branch 스킬을 사용합니다."
- **필수 하위 스킬:** `superpowers:finishing-a-development-branch`를 사용한다.
- 해당 스킬을 따라 테스트를 검증하고, 선택지를 제시하고, 선택된 작업을 실행한다.

## 멈추고 도움을 요청해야 할 때

**다음 상황에서는 즉시 실행을 멈춘다:**

- 블로커에 부딪힘: 누락된 의존성, 테스트 실패, 불명확한 지시
- 계획에 시작을 막는 치명적 공백이 있음
- 지시를 이해하지 못함
- 검증이 반복해서 실패함

**추측하지 말고 명확히 물어본다.**

## 이전 단계로 돌아가야 할 때

**다음 상황에서는 검토 단계(1단계)로 돌아간다:**

- 파트너가 피드백을 반영해 계획을 업데이트함
- 근본적인 접근 방식을 다시 생각해야 함

**블로커를 억지로 밀고 나가지 않는다.** 멈추고 물어본다.

## 기억할 것

- 먼저 계획을 비판적으로 검토한다.
- 계획 단계를 정확히 따른다.
- 검증을 건너뛰지 않는다.
- 계획에서 지시하는 스킬을 참조한다.
- 막히면 멈춘다. 추측하지 않는다.
- 명시적인 사용자 동의 없이 `main`/`master` 브랜치에서 구현을 시작하지 않는다.

## 통합

**필수 워크플로우 스킬:**

- **superpowers:using-git-worktrees** - 격리된 작업공간을 보장한다. 새로 만들거나 기존 작업공간을 확인한다.
- **superpowers:writing-plans** - 이 스킬이 실행할 계획을 작성한다.
- **superpowers:finishing-a-development-branch** - 모든 작업 후 개발을 완료한다.
