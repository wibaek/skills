# ADR Writer

Paulo Merson 스타일의 간결한 ADR을 작성한다. 결정 자체보다 “왜 그렇게 결정했는지”를 명확히 남기는 데 집중한다.

## 워크플로우

1. 기존 관례를 먼저 확인한다.
   - `docs/adr`, `docs/decisions`, `adr`, `architecture` 아래의 ADR 파일을 찾는다.
   - 기존 파일이 있으면 번호, 파일명, 상태 값, 문서 언어, 제목 형식을 따른다.
   - 관례가 없으면 `docs/adr/NNNN-kebab-case-title.md`를 기본값으로 사용한다.

2. 결정 정보를 정리한다.
   - 날짜: 기존 ADR 관례가 없으면 현재 날짜를 `YYYY-MM-DD` 형식으로 기록한다.
   - 상태: 기본값은 `제안됨`; 사용자가 확정된 결정이라고 말하면 `승인됨`을 사용한다.
   - 문제 상황: 어떤 문제나 제약 때문에 결정이 필요한가.
   - 결정: 무엇을 하기로 했는가.
   - 근거: 왜 이 선택이 적합한가.
   - 대안: 검토했지만 선택하지 않은 대안은 무엇인가.
   - 결과: 이 결정으로 얻는 것과 감수하는 것은 무엇인가.

3. 정보가 부족하면 최소한으로 질문한다.
   - 결정 자체나 상태가 불명확하면 질문한다.
   - 근거나 대안이 일부 부족해도, 코드베이스와 사용자 설명에서 합리적으로 추론할 수 있으면 초안을 작성하고 가정을 명시한다.

4. Paulo Merson 구조로 작성한다.
   - 제목 바로 아래에 `날짜`와 `상태`를 메타데이터로 적는다.
   - 필수 섹션은 `문제 상황`, `결정`, `근거`, `결과`다.
   - 상세 대안 비교표가 필요할 정도로 복잡하면 MADR이 더 적합하다고 짧게 언급하되, 사용자가 요청하지 않으면 Paulo Merson 구조를 유지한다.

5. 완료 전에 검토한다.
   - 섹션 순서가 맞는지 확인한다.
   - 제목 바로 아래에 날짜와 상태가 메타데이터로 있는지 확인한다.
   - 날짜가 `YYYY-MM-DD` 형식인지 확인한다.
   - 문제 상황이 결정 위에 있고, 결정이 필요한 이유를 설명하는지 확인한다.
   - 결정과 결과가 서로 모순되지 않는지 확인한다.
   - 문서 언어가 일관적인지 확인한다.
   - 기술명, 제품명, 고유명사는 원문 표기를 유지한다.

## 문서 언어

- 사용자가 한국어로 요청했거나 repository 문서가 한국어면 한국어로 작성한다.
- repository 문서가 영어면 영어로 작성한다.
- 한 문서 안에서 제목은 영어, 본문은 한국어, 평가 표현은 영어처럼 섞지 않는다.
- 기술명과 제품명은 번역하지 않는다. 예: `PostgreSQL`, `AWS RDS`, `Alembic`.

## 기본 템플릿

```markdown
# ADR NNNN: [결정 제목]

날짜: YYYY-MM-DD
상태: [제안됨 | 승인됨 | 대체됨 | 폐기됨]

## 문제 상황

[문제가 되는 상황과 결정이 필요한 이유를 1-3문단으로 설명한다.]

## 결정

[무엇을 하기로 했는지 명확하게 쓴다.]

## 근거

[왜 이 결정이 적합한지 설명한다.]

검토했지만 선택하지 않은 대안은 다음과 같다.

- [대안 1]: [선택하지 않은 이유]
- [대안 2]: [선택하지 않은 이유]

이 결정은 다음 가정을 전제로 한다.

- [가정 1]
- [가정 2]

## 결과

[결정 이후 필요한 운영, 구현, 유지보수 변화를 요약한다.]

긍정적 결과는 다음과 같다.

- [긍정적 결과 1]
- [긍정적 결과 2]

부정적 결과는 다음과 같다.

- [부정적 결과 1]
- [부정적 결과 2]
```

## 작성 기준

- ADR 하나에는 결정 하나만 담는다.
- 결정문은 모호한 방향성이 아니라 실행 가능한 선택으로 쓴다.
- 근거에는 선택 이유, 버린 대안, 중요한 가정, 제약, 실험 결과를 포함할 수 있다.
- 결과에는 좋은 점과 비용을 모두 남긴다.
- 미래에 할 수도 있는 일을 결정한 것처럼 쓰지 않는다.
- 아직 검증되지 않은 판단은 사실처럼 쓰지 말고 가정이나 리스크로 표시한다.
- 코드나 설정 변경과 함께 작성한다면 ADR 내용이 실제 변경과 일치하는지 확인한다.
